## 🚀 Быстрый старт

### Установка локально

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/32mx32/instabot.git
cd instabot
```

2. **Создайте виртуальное окружение:**
```bash
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows
```

3. **Установите зависимости:**
```bash
pip install -r requirements.txt
```

4. **Настройте переменные окружения:**
```bash
cp .env.example .env
nano .env  # или любой другой редактор
```

Заполните файл `.env`:
```env
TELEGRAM_BOT_TOKEN=your_bot_token_from_@BotFather
APIFY_API_KEY=your_apify_api_key
# или несколько ключей через запятую:
# APIFY_API_KEYS=apify_api_key_1,apify_api_key_2,apify_api_key_3
DELETE_AFTER_SEND=true
```

5. **Запустите бота:**
```bash
python main.py
```

## 🐳 Docker развертывание

### Локально с Docker


## 📖 Использование

### Команды бота

- `/start` - Начать работу с ботом и посмотреть возможности

### Как скачать контент

1. Откройте Instagram
2. Найдите нужный Reels или пост
3. Нажмите **Поделиться** → **Копировать ссылку**
4. Отправьте ссылку боту в Telegram

Бот поддерживает следующие типы ссылок:
```
https://instagram.com/reel/ABC123...
https://instagram.com/p/ABC123...
https://instagram.com/tv/ABC123...
```

### Дополнительные возможности

**Комментарии к постам:**
- Если у поста есть комментарий, появится кнопка "📄 Показать комментарий"
- Нажмите на нее, чтобы увидеть текст в всплывающем окне

**Повторная попытка:**
- Если скачивание не удалось, нажмите кнопку "🔄 Повторить"
- Бот сделает еще 3 попытки

## 🔧 Конфигурация

### Переменные окружения (.env)

| Переменная | Описание | Обязательна | Значение по умолчанию |
|------------|----------|-------------|----------------------|
| `TELEGRAM_BOT_TOKEN` | Токен бота от [@BotFather](https://t.me/BotFather) | ✅ Да | - |
| `APIFY_API_KEY` | Один API ключ от [Apify](https://console.apify.com/settings/integrations), старый совместимый формат | ✅ Да* | - |
| `APIFY_API_KEYS` | Несколько API ключей Apify через запятую для автоматического переключения при лимитах | ✅ Да* | - |
| `DELETE_AFTER_SEND` | Удалять файлы после отправки | ❌ Нет | `true` |

`*` Нужно задать хотя бы одну из переменных: `APIFY_API_KEY` или `APIFY_API_KEYS`.

Пример с несколькими Apify аккаунтами:

```env
APIFY_API_KEYS=apify_api_key_1,apify_api_key_2,apify_api_key_3
```

Если Apify вернет ошибку лимита, квоты, оплаты, авторизации или rate limit по текущему ключу, бот временно отключит этот ключ и попробует следующий.

### Настройки в коде

В `main.py` можно изменить:

```python
MAX_REQUESTS_PER_MINUTE = 5     # Лимит запросов на пользователя
MAX_CACHE_SIZE = 1000           # Размер кэша комментариев/URL
MAX_FILE_CACHE_SIZE = 50        # Максимум файлов в кэше
CACHE_TTL = 3600                # Время жизни кэша (1 час в секундах)
```

**Рекомендации:**
- Увеличьте `MAX_FILE_CACHE_SIZE` если у вас много памяти и частые повторы одних постов
- Уменьшите `CACHE_TTL` если нужны всегда свежие версии постов
- Увеличьте `MAX_REQUESTS_PER_MINUTE` если бот используется одним человеком

## 🚀 Новые возможности

### 🗂️ Кэширование файлов
Бот автоматически кэширует скачанные файлы на **1 час**. Если кто-то запросит тот же пост снова в течение часа, файл будет отправлен мгновенно из кэша!

**Преимущества:**
- ⚡ Мгновенная отправка популярных постов
- 💰 Экономия API запросов к Apify
- 🌐 Меньше нагрузки на Instagram
- 📊 До 50 файлов в кэше одновременно

**Индикация:**
```
✅ Файл найден в кэше!  ← Файл отправлен из кэша
⏳ Скачиваю контент...  ← Скачивание с Instagram
```

### 🔍 Детальный анализ ошибок
Бот теперь распознает типы ошибок и дает точные рекомендации:

| Ошибка | Что означает | Что делать |
|--------|-------------|------------|
| ⏱️ Timeout | Сервер долго отвечает | Повторить через минуту |
| 🌐 Connection | Проблемы с интернетом | Проверить соединение |
| 🔒 401 Unauthorized | Instagram заблокировал | Подождать 5-10 минут |
| ❌ 404 Not Found | Пост удален/не найден | Проверить ссылку |
| 🚫 429 Rate Limit | Слишком много запросов | Подождать 10-15 минут |

### 🎯 Type Hints
Весь код покрыт type hints для лучшей поддержки IDE и предотвращения ошибок:
```python
async def download_reel_with_retry(
    post_url: str, 
    max_attempts: int = 3, 
    status_message: Optional[Message] = None
) -> Tuple[Optional[str], Optional[str], Optional[str]]:
    ...
```

## ⚠️ Ограничения

| Ограничение | Значение |
|-------------|----------|
| Rate limit | 5 запросов/минуту на пользователя |
| Максимум попыток | 3 автоматических + ручные повторы |
| Кэш URL | 1000 записей |
| Кэш комментариев | 1000 записей |
| Кэш файлов | 50 файлов, TTL 1 час |

**Не поддерживается:**
- ❌ Instagram Stories (временные истории)
- ❌ Приватные аккаунты
- ❌ Защищенный контент
- ❌ Альбомы с несколькими фото/видео (пока)

## 🐛 Troubleshooting

### Бот не отвечает
1. Проверьте, что бот запущен: `docker-compose ps` или `ps aux | grep python`
2. Проверьте логи: `docker-compose logs` или `tail -f bot.log`
3. Проверьте токены в `.env`

### Ошибка "Rate limit exceeded"
- Подождите 1 минуту и попробуйте снова
- Это защита от спама (максимум 5 запросов/минуту)

### Не скачивается контент
1. Проверьте, что ссылка правильная
2. Убедитесь, что аккаунт не приватный
3. Попробуйте кнопку "🔄 Повторить"
4. Проверьте Apify API ключ

### Apify ошибка "Please wait a few minutes"
- Instagram временно заблокировал IP Apify
- Подождите 5-10 минут и попробуйте снова
- Используйте кнопку "🔄 Повторить"
