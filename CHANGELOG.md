# Changelog - Instagram Telegram Bot

## [v2.1.0] - 2025-10-09

### 🆕 Новые возможности
- **Кэширование файлов**: Автоматическое кэширование скачанных файлов на 1 час (до 50 файлов)
- **Type Hints**: Полное покрытие всех функций type hints для Python 3.10+
- **Детальный анализ ошибок**: Распознавание типов ошибок (timeout, 401, 404, 429) с конкретными рекомендациями
- **Версионирование**: Добавлена версия бота в команды /start и /help

### ⚡ Производительность
- Мгновенная отправка популярных постов из кэша
- Экономия API запросов к Apify
- Меньше нагрузки на Instagram API
- Умная очистка кэша по TTL и размеру

### 🔍 Улучшения
- Rate limiting (5 запросов/минуту на пользователя)
- Команды /help, /stats с подробной информацией
- Индикатор прогресса (попытка 1/3, 2/3, 3/3)
- Поддержка обычных постов (/p/) и IGTV (/tv/)
- Логирование в файл bot.log
- Статистика использования с процентом успеха
- Механизм повторных попыток (3 автоматических + кнопка "Повторить")

### 📖 Документация
- Полностью переписан README.md
- Добавлены примеры использования
- Таблица типов ошибок
- Инструкции по настройке кэша
- Badges с версией и статусом

### 🧹 Очистка кода
- Удалены неиспользуемые файлы (config.example.py, .github/workflows)
- Улучшена структура проекта
- Добавлены комментарии к функциям

## [v2.0.0] - 2025-08-03

### 🔐 Безопасность
- **КРИТИЧНО**: Удален хардкод токенов из кода
- Токены теперь загружаются только из .env файла
- Добавлена обязательная проверка наличия токенов при запуске
- Улучшена валидация токенов в install.sh

### ⏱️ Надежность
- Добавлены таймауты (30 сек) для всех HTTP запросов
- Реализован retry механизм с экспоненциальной задержкой для Apify запросов
- Улучшена обработка сетевых ошибок (Timeout, ConnectionError, RequestException)
- Все три метода скачивания Apify теперь используют retry механизм

### 💾 Управление памятью
- Добавлено ограничение размера кэша комментариев (1000 записей)
- Реализована автоматическая очистка кэша при превышении лимита
- Оптимизировано логирование комментариев (обрезка длинных текстов)

### 🛠️ Улучшения развертывания
- Улучшен install.sh с валидацией токенов и проверкой статуса
- Обновлен quick-update.sh с лучшим логированием и проверками
- Добавлена проверка работоспособности контейнера после запуска
- Улучшены сообщения об ошибках и статусе

### 🔧 Технические улучшения
- Добавлен импорт `time` для retry механизма
- Улучшена обработка определения типа файлов с таймаутом
- Оптимизирована структура кода для лучшей читаемости

## Инструкции по обновлению

### Для существующих установок:
```bash
./quick-update.sh
```

### Для новых установок:
```bash
./install.sh
```

### Проверка статуса:
```bash
docker compose ps
docker compose logs -f
```

## Совместимость
- Python 3.10+
- Docker & Docker Compose
- Все существующие функции сохранены
- Обратная совместимость с существующими .env файлами
